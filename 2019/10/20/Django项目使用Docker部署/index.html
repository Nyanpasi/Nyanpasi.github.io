使用Docker部署Django项目

Posted on 2019-10-20 | 该文章被访问 次

Django使用Mysql作为数据库, Redis作为缓存, uWSGI作为WSGI Server, 并使用Nginx作为反向代理, 所以一共需要四个容器:

  1. Mysql容器, 提供数据库服务
  2. Redis容器, 提供缓存服务
  3. uWSGI容器, 作为WSGI Server
  4. Nginx容器, 作为反向代理, 转发动态请求, 处理静态资源

创建网络

所有容器运行在内部网络上, 通过容器间通信互相访问, 只有Nginx映射至host的80和443端口, 以供外部访问.

内部网络命名为django_project, 之后所有容器都会运行在这个网络上.

正式启动uwsgi容器

在启动命令中取消端口映射, 只允许以容器间通信的方式进行访问.

uWSGI服务器按照dockerfile中给出命令, 以socket模式正式启动, 此时uWSGI服务器只接收uwsgi协议而不是http协议的请求.

Nginx容器

Nginx作为反向代理, 主要提供三个功能

  1. 返回静态网页
  2. 返回静态资源
  3. 作为反向代理, 以uwsgi协议将动态请求转发至uWSGI服务器.

第一项可能是静态的首页文件, 或是由前端框架生成的静态页面, 我这里是vue框架build之后的index.html.
第二项则是Django项目的static和media文件夹中的资源, 取决于django项目的设置.
第三项则是所有非静态资源请求, 都要转发至之前的uwsgi容器中的uWSGI服务器进行处理.
