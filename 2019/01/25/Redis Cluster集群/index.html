Redis Cluster

Redis Sentinel 着眼于高可用, 但Redis实例的容量仍然受限于单机的内存容量.

Cluster提供将多个redis实例作为一个整体, 根据slot的概念, 将键分配在多个redis实例中, 这样redis的容量就不再受限于单个机器的内存.

每个被分配了一部分槽的redis实例称为一个节点, 在建立集群后, 为每个节点设置主从复制, 并在master出错时, 将slave提升为master.

Cluster的主从切换并不需要使用Sentinel, 而是通过多个节点之间的通信协商.

创建集群的流程为

  1. 创建多个Redis节点
  2. 创建集群
  3. 使用客户端连接

创建多个redis节点

每个节点是一个redis实例, 官方给出作为一个节点的最小配置项如下:

cluster-config-file 指定保存节点信息状态的文件, 我们只需要为这个文件指名, 但不需要手动管理该文件, 会由Cluster进行创建和随时更新.
cluster-node-timeout 指定在一个节点在无响应多少毫秒后视为不可达.

访问集群

访问集群的操作与访问单个redis实例不同, 因为每个节点都只负责一部分槽.

可以访问任何一个节点, 但除非所读写的键正好在该节点所分配的槽的范围内, 否则节点会返回一个Moved错误, 并给出该槽被分配在节点的地址.

需要一个封装良好的客户端, 来进行对集群的操作, 例如python的redis-py-cluster包.
