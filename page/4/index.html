Nginx更改root无效

Posted on 2019-07-23

之前一直用Docker部署Nginx, 从来没有遇见过这个问题.

这次因为要申请Let's Entrypt的SSL证书, 所以不得不在宿主机上构建一个能够返回静态文件的网站.

使用Nginx返回文件需要设定目录, Nginx的默认根目录为/usr/share/nginx/html, 而我想更改这个目录

打算更改为/nginx/home, 于是修改nginx.conf中的root选项, 不过之后却一直404.

打开错误日志

显示Nginx一直在默认目录下查找文件, 对root的更改未生效.

原因是因为设置项中包含了这一项

该项所指定的包含项中有default.conf, 而其中设置了root, 也就是更改的设置又被该文件给覆盖了.

我实在无法理解这样设计的意义是什么, 不过解决方案倒是很简单, 直接将include行注释掉即可.

Python Selectors模块

Posted on 2019-07-22

selectors模块提供高度封装和高效的I/O复用, 建立在底层的I/O复用模块select之上.

在绝大多数情况下, 用户应当使用selector而非select.

类的继承关系

BaseSelector
+– SelectSelector
+– PollSelector
+– EpollSelector
+– DevpollSelector
+– KqueueSelector

events是一个位掩码,用于指定某个文件对象应当监听什么事件, 并可与以下的常量联合使用

Constant Meaning
EVENT_READ 可读
EVENT_WRITE 可写

核心类

selectors.SelectorKey类

一个SelectorKey是一个命名元组, 用于联系文件、文件的描述符、选定的事件掩码、以及相关联的数据.

  1. fileobj 已登记的文件对象.
  2. fd 已登记文件对象的文件描述符 file descriptor.
  3. events 该文件正在等待的事件.
  4. data 可选的用于储存相关信息的变量, 例如可以用来储存当该事件发生后应当调用的函数.

selectors.BaseSelector类

一个BaseSelector用于等待多个文件的I/O就绪事件, 它实现或定义了以下方法:

方法 作用
register(fileobj, events, data=None) 登记一个文件对象, 监听其I/O事件.
unregister(fileobj) 取消对某个文件对象的监听.
modify(fileobj, events, data=None) 更改文件对象的监听事件或相关数据.
select(timeout=None) 尝试获得可用文件, 返回一个(key, events)元组的列表.
close() 关闭selector
get_key(fileobj) 返回与该文件关联的SelectorKey实例
get_map() 返回一个文件对象和其关联的SelectorKey实例的mapping对象

BaseSelector对于与平台相关的一些方法只定义了其抽象实现, 而具体的实现则在以下的几个类中:

具体实现 底层IO复用模型
selectors.SelectSelector 基于slect实现
selectors.PollSelector 基于poll实现
selectors.EpollSelector 基于epoll实现
selectors.DevpollSelector 基于devpoll实现
selectors.KqueueSelector 基于kqueue实现

在绝大多数情况下都应当使用 selectors.DefaultSelector 类来自动获得当前平台合适的Selector类
